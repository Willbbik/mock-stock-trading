
## command 폴더
- 여러 도메인을 조합하여 특정 기능을 수행하는 서비스 계층
- facade 패턴

#### 예시
- 회원 가입 with (계좌 생성, 메일 발송, 포인트 적립) 
- 주문 with (주문 생성, 결제 생성, 재고 차감) 


#### 생성 규칙
- 메소드명: ooo+CommandService ex) OrderCommandService
- 클래스는 가능하면 큰 범위 (회원, 결제 등)를 기준으로 생성
- 메소드가 너무 많아지는 경우 세부 기능별로 서비스 클래스로 분리

## query 폴더
단순 데이터 조회가 필요한 경우 사용하는 서비스 계층

#### 예시
- 도메인의 정책 및 유효성 검사, 복잡한 계산, ID 또는 코드 생성

#### 생성 규칙
- 메소드명: ooo+QueryService ex) OrderQueryService
- 트랜잭션이 필요 없는 순수 도메인 로직이 필요한 경우 (CUD 필요시 command 사용)
- 다른 도메인의 repository 사용하지 않기 (필요 시 다른 도메인 서비스 호출)
